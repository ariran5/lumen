## TL;DR

| ID | Фича |
|---|---|
| P4.2.c | `animated(initial)` AnimatedValue: `.set/.animateTo/.stop`, off-main CABasicAnimation/CASpringAnimation |

JS-API:
```ts
const x = animated(0)
const y = animated(0)

View({
  transform: {translateX: x, translateY: y},
  onPan: (e) => {
    if (e.state === 'start')  { x.stop(); y.stop() }   // catch in flight
    if (e.state === 'changed'){ x.set(e.dx); y.set(e.dy) }
    if (e.state === 'ended')  {
      x.animateTo(0, {easing: 'spring'})
      y.animateTo(0, {easing: 'spring'})
    }
  },
})
```

Реализация финиширована, build clean, tsc clean, симулятор поднимает DragLab с обоими шариками (blue panning + orange snap-back).

## Архитектура

**Главный принцип:** значение хранится **на native стороне** в `AnimationManager.nodes[id].current`. JS-side AnimatedValue — лишь facade, посылает команды через `lumen._animValue.{create, set, animateTo, stop, current, release, resetAll}`. JS-loop не участвует в кадрах — анимации крутятся на render-сервере (backboardd) через `CABasicAnimation` / `CASpringAnimation`. Даже если JS залип на 200ms, движение не дрогнет.

### Поток данных

1. **JS-side** ([CoreFramework.swift](../Sources/LumenRuntime/CoreFramework.swift)):
   ```js
   function AnimatedValue(initial) {
     this.__anim = _nextAnimId()
     this._current = initial
     lumen._animValue.create(this.__anim, initial)
   }
   ```
   AnimatedValue имеет `__anim: id` (sentinel) + `_current` (для JS-side reads). Прототип методов: `set / animateTo / stop / current / release`.

2. **Парсинг style** ([RenderNode.swift](../Sources/LumenRuntime/RenderNode.swift)):
   `parseAnimOrDouble(any)` возвращает `(value: Double?, animId: Int?)`. Если `any` — число → static. Если `{__anim: id, ...}` → animated binding. Применяется к `transform.{translateX, translateY, scale, scaleX, scaleY, rotate}` и `opacity`.

3. **Renderer** ([Renderer.swift](../Sources/LumenRuntime/Renderer.swift)) — `applyVisualStyle`:
   - Если `style.hasAnimBindings` → `AnimationManager.bindLayer(layer, animIds, staticParts...)`. AnimationManager сразу композирует и применяет transform + opacity.
   - Иначе → `unbindLayer(layer)` (на случай если был animated раньше) + прямой apply как раньше.

4. **AnimationManager** ([AnimationManager.swift](../Sources/LumenRuntime/AnimationManager.swift)):
   - `nodes: [Int: AnimNode]` — по animId хранит `current: Double` + `layerIds: Set<ObjectIdentifier>`
   - `layers: [ObjectIdentifier: LayerState]` — по CALayer хранит `[Property: animId]` mapping + static fallback'и
   - `composedTransform(state)` — для каждого Property берёт `nodes[animId].current` если есть animId, иначе static; собирает `CATransform3D`.
   - `composedOpacity(state)` — то же для opacity.

### `.set(v)`

1. Обновить `node.current = v`
2. Для каждого bound `layer`: снять in-flight CABasicAnimation (`removeAnimation(forKey: "lumen-anim-{id}-t/o")`), композировать новый transform/opacity, применить мгновенно с `CATransaction.setDisableActions(true)`.

### `.animateTo(v, {duration, easing})`

1. Обновить `node.current = v` (= target)
2. Для каждого bound layer:
   - Считать `presentation()?.transform` / `.opacity` — это «откуда лететь» (если уже анимация в полёте — берётся реальное положение)
   - Установить layer.transform / .opacity = target (модель) — без implicit-анимаций
   - Наложить `CABasicAnimation(keyPath: "transform")` / `("opacity")` от presentation → new

   Если easing == `'spring'` → `CASpringAnimation` (`stiffness=200, damping=18, mass=1`), duration = settlingDuration.

### `.stop()`

Самый аккуратный момент — взять `presentation()` и **сделать его новой моделью** для AnimNode + applied к layer'у. Декомпозиция матрицы:
- `translateX` ← `transform.m41`
- `translateY` ← `transform.m42`
- `scale, scaleX` ← `sqrt(m11² + m12²)`
- `scaleY` ← `sqrt(m21² + m22²)`
- `rotate` ← `atan2(m12, m11)`

Декомпозиция корректна для 2D-аффинных трансформов (наш случай). После `.stop()`:
1. `node.current = presentationValue` (single source of truth)
2. На каждом bound layer: `removeAnimation(forKey: ...)`, перекомпозить, applied — `CATransaction.disable`

Это даёт **clean freeze** — layer стоит ровно там же где визуально был. Никакого рывка от model = target → presentation = inFlight → freeze = target.

### Lifecycle

- **`removeMountTree`** → `AnimationManager.unbindLayer(mount.layer)`. Это критично — без него removed layer'ы остаются в `layers` map с `weak layer = nil`, накапливают мусор.
- **Hot reload** → `AnimationManager.shared.reset()` в [FastAppHost.performReload](../Sources/LumenShell/FastAppHost.swift) перед `setupEngine()`. Иначе новые AnimatedValue стартуют id с 1 и сталкиваются со stale-записями мёртвого контекста.

## API на стороне @lumen/types

В [packages/lumen-types/index.d.ts](../packages/lumen-types/index.d.ts):

```ts
type AnimatableNumber = number | AnimatedValue

interface TransformProps {
  translateX?: AnimatableNumber
  translateY?: AnimatableNumber
  scale?: AnimatableNumber
  scaleX?: AnimatableNumber
  scaleY?: AnimatableNumber
  rotate?: AnimatableNumber
}
interface VisualProps {
  opacity?: AnimatableNumber
  // ...
}

type Easing = 'linear' | 'easeIn' | 'easeOut' | 'easeInOut' | 'spring'

interface AnimateOptions {
  duration?: number       // ms, игнорируется для spring
  easing?: Easing
}

interface AnimatedValue {
  readonly __anim: number
  current(): number
  set(value: number): void
  animateTo(value: number, opts?: AnimateOptions): void
  stop(): number          // returns the captured presentation value
  release(): void         // редко нужен; reset на hot-reload автоматический
}

function animated(initial?: number): AnimatedValue
```

`AnimatableNumber` — union `number | AnimatedValue` — даёт type-safe binding в style. tsc на DragLab и HN clean.

## Demo (Snap-back ball)

[Examples/DragLab/index.ts](../Examples/DragLab/index.ts) — добавлен `SnapArea()`:

```ts
const snapX = animated(0)
const snapY = animated(0)
let dragStartX = 0
let dragStartY = 0

View({
  height: 180,
  onPan: (e) => {
    if (e.state === 'start') {
      snapX.stop(); snapY.stop()         // catch mid-flight
      dragStartX = snapX.current()
      dragStartY = snapY.current()
      snapX.set(dragStartX); snapY.set(dragStartY)
    } else if (e.state === 'changed') {
      snapX.set(dragStartX + e.dx)
      snapY.set(dragStartY + e.dy)
    } else if (e.state === 'ended' || e.state === 'cancelled') {
      snapX.animateTo(0, {easing: 'spring'})
      snapY.animateTo(0, {easing: 'spring'})
    }
  },
},
  // ...
  View({
    width: 60, height: 60, borderRadius: 30,
    backgroundColor: '#F59E0B',
    transform: {translateX: snapX, translateY: snapY},
  }),
)
```

**Поведение:**
1. Тащишь оранжевый шарик — он едет за пальцем (через `.set` на каждый pan-changed)
2. Отпускаешь — `animateTo(0, {easing: 'spring'})` запускает spring, шарик летит обратно
3. Хватаешь летящий шарик — `pan start` → `.stop()` фиксирует presentation-позицию и шарик сразу слушается пальца с того места где визуально был

Pan rebase через `dragStartX/Y` нужен потому что UIPan.translation начинается с 0 на каждый цикл; на `start` мы запоминаем «куда уже улетел» шарик через `.current()` и в `changed` считаем абсолют.
