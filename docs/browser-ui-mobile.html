An alternative renderer that treats the browser as a real native surface.

by aria-n 4 min read

For two decades the web has been shaped by a single rendering pipeline. HTML, CSS and JavaScript all funnel into the same compositor — the page. We took a step back and asked: what happens if you let the platform render itself?

The result is Lumen — a browser that treats a webpage as a description, not a pixel buffer. Text flows through the system text engine. Images decode through the platform. Scrolling and accessibility behave the way the user already expects from the rest of their device.

"We didn't want a browser that looks like an app. We wanted an app that happens to be a browser."

The trade-off, of course, is fidelity — and that is exactly the conversation we wanted to start.
