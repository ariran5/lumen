An alternative renderer that treats the browser as a real native surface.

by aria-n 4 min read 284 points · 142 comments

For two decades the web has been shaped by a single rendering pipeline. HTML, CSS and JavaScript all funnel into the same compositor — the page. We took a step back and asked: what happens if you let the platform render itself? Not by mapping DOM elements to native widgets, but by exchanging the page metaphor for a layout graph the OS already understands.

The result is Lumen — a browser that treats a webpage as a description, not a pixel buffer. Text flows through the system text engine. Images decode through the platform. Scrolling, accessibility and input all behave the way the user already expects from the rest of their device.

"We didn't want a browser that looks like an app. We wanted an app that happens to be a browser."

The trade-off, of course, is fidelity — and that is exactly the conversation we wanted to start. The early build supports a focused subset of the modern web, with extensions written in plain SwiftUI. If you're curious, the source is open and the design notes are linked below.

This thread is an open invitation: poke at the assumptions, ask the awkward questions, and tell us where the metaphor breaks down.
